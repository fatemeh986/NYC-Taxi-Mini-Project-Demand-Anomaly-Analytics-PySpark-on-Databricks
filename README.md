I built a small lakehouse-style pipeline in Databricks using PySpark, starting from a raw NYC Yellow Taxi Parquet dataset. The project follows a Bronze → Silver → Gold structure: Bronze keeps the raw schema, Silver enforces data quality rules (e.g., non-negative distance/fare, reasonable passenger counts, and pickup time < dropoff time) and routes invalid rows into a quarantine dataset with an invalid_reason, and Gold produces analytics-ready tables.

In the Gold layer, I generated aggregated views for common reporting needs: daily KPIs (trips/day, average distance/fare, revenue), an origin–destination zone heatmap (trips by pickup/dropoff location IDs), peak-hour demand (trips by hour and day of week), and payment-type distribution (trip share by payment method). These curated outputs are designed to be directly usable for dashboards and as a foundation for lightweight ML modeling in the next step.
